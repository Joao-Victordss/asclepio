Mastite Bovina

Sistema em Python para classificar o risco de mastite em vacas leiteiras usando dados numéricos de sensores do úbere e temperatura, entregando uma resposta clara: "Mastite" ou "Saudável", com as probabilidades de cada classe. No conjunto de teste, o modelo atingiu 99,5% de acurácia.

O projeto

O que é este sistema?

Tecnologia aplicada à bovinocultura leiteira

A mastite é uma inflamação da glândula mamária e uma das doenças mais frequentes e onerosas na bovinocultura leiteira. Ela causa redução na produção de leite, descarte de leite contaminado, custos elevados com tratamento veterinário e, em casos severos, pode levar ao descarte do animal.

Este sistema utiliza dados numéricos coletados por sensores IoT posicionados no úbere, junto com a temperatura corporal, para, por meio de um modelo de Machine Learning (Aprendizado de Máquina), apoiar a identificação precoce do risco de mastite.

Processo

Como funciona (fluxo completo)

Cada etapa do pipeline (fluxo de processamento de dados) explicada

1

Entrada de dados via CSV

O sistema recebe um arquivo CSV (arquivo de dados em formato tabular) com um identificador do animal (coluna ID) e as variáveis numéricas coletadas pelos sensores:

  • Months_after_giving_birth: meses pós-parto
  • IUFL, EUFL, IUFR, EUFR: úbere frontal
  • IURL, EURL, IURR, EURR: úbere traseiro
  • Temperature: temperatura corporal
2

Preparação da base

Antes do treinamento, a base de dados passa por um tratamento para ficar consistente:

  • Padronização dos nomes de colunas
  • Conversão das variáveis para formato numérico
  • Remoção de linhas incompletas ou inválidas
  • Padronização do rótulo (classe) para um formato único
3

Balanceamento de classes

Em problemas de saúde animal, é comum ter desbalanceamento, com mais casos saudáveis do que doentes, ou vice-versa. Para reduzir viés, o projeto aplica uma técnica de oversampling (superamostragem) para equilibrar as classes e melhorar a aprendizagem do modelo.

4

Treinamento com Random Forest

Com a base tratada e balanceada:

  • Divisão em treino e teste, ou split (80% / 20%)
  • Treinamento do Random Forest (Floresta Aleatória)
  • Cálculo de acurácia, matriz de confusão e relatório de classificação
  • Sensibilidade (detectar mastite) e especificidade (reconhecer saudáveis)
5

Geração do artefato do modelo

Após o treinamento, o modelo é salvo em arquivo .pkl.gz (arquivo compactado) para facilitar o carregamento na API FastAPI e no uso da interface web.

6

API FastAPI + interface web

A aplicação web permite:

  • Upload (envio) de um CSV no formato esperado
  • Entrada manual dos dados diretamente pelo navegador
  • Validação dos dados (tipos numéricos e campos obrigatórios)
  • Resultado em tabela com classe prevista e probabilidades

Limitações e aviso importante

Este sistema é um apoio computacional, não um diagnóstico definitivo. Ele não substitui a avaliação clínica e a conduta de um médico veterinário. O modelo atual é propositalmente objetivo e direto, usando Random Forest (Floresta Aleatória), e pode ser evoluído futuramente com comparações com outros algoritmos e técnicas de explicabilidade (formas de entender por que o modelo tomou determinada decisão).
